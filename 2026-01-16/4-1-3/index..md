## 4. ⚖️ 4xx(클라이언트 에러) vs 5xx(서버 에러) 차이점 ⚖️

### 🔴 4xx (Client Error)

#### 클라이언트 (사용자, 프론트엔드)

- 원인 : 주소 오타 났거나, 잘못 입력했을 때
  : 로그인을 안 했는데 회원 전용 페이지를 달라고 요청 할때
  : 존재하지 않는 데이터를 달라고 요청했을 때

- 해결 : 요청 보내는 코드나 입력값(URL, 파라미터)을 수정

### 💥 5xx (Server Error)

#### 서버 (백엔드, DB)

- 원인 : 서버 내부 코드가 터졌을때 (버그).
  : 데이터베이스(DB) 연결이 끊겼을때
  : 서버 컴퓨터 전원이 꺼지거나, 이용자가 몰려 트래픽이 폭주했을 때

- 해결 : 서버 개발자에게 연락해서 해결해 달라고 요청 (직접 할 수 있는 게 없음)

## 🔑 주요 상태 코드 🔑

#### ✅ 2xx : 성공

- 200 OK : 에러 없이 요청이 완벽하게 처리된 기본적인 성공 상태
- 201 Created : 새로운 데이터(리소스)가 성공적으로 생성 (ex - 회원가입, 글쓰기 등..)
- 204 No Content : 요청은 성공적으로 수행했지만, 응답해줄 데이터 본문(Body)이 없음

#### ❓ 4xx : 클라이언트 에러

- 400 Bad Request : 잘못된 요청 (입력 데이터의 형식이 틀렸거나 필수 값이 누락됨)
- 401 Unauthorized : 로그인 필요 (신분 증명 실패).
- 403 Forbidden : 로그인은 했지만 권한이 없음 (관리자 페이지 접근 등).
- 404 Not Found : URL 주소가 틀렸거나 이미 삭제된 게시물.

#### 💥 5xx: 서버 에러

- 500 Internal Server Error : 서버 내부 오류 (웹 서버가 요청을 처리하는 과정에서 알 수 없는 에러 발생).
- 503 Service Unavailable : 서버가 일시적으로 요청을 처리할 수 없음 (서버 점검 중이거나, 접속자가 너무 많아 폭주했을 때).
