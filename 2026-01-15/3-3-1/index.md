## 1. 공개 API 접속 및 JSON 구조 분석

- 분석 대상 필드: `id`, `name`, `title`, `completed`

👉 접속 URL: https://jsonplaceholder.typicode.com/todos/1

```json
{
  "userId": 1,
  "id": 1,
  "title": "delectus aut autem",
  "completed": false
}
```

### 📋 JSON 필드 분석표

| <big>필드명 (Key)</big> | <big>데이터 타입</big> |             <big>의미 및 설명</big>             |
| :---------------------: | :--------------------: | :---------------------------------------------: |
|         userId          |         Number         |             작성한 사용자의 고유 ID             |
|           id            |         Number         |             할 일 데이터 고유 번호              |
|          title          |         String         |               할 일 내용 (텍스트)               |
|        completed        |        Boolean         | 할 일이 끝났는지 여부 (true=완료, false=미완료) |

## 2. Request vs Response 차이점

### 🚀 Request (요청)

- 클라이언트 ➡️ 서버  
   ➔ 필요한 데이터를 달라고 서버에 시키는 것
  > 🗣️ 클라이언트: "게시글 목록 주세요"

### 📦 Response (응답)

- 서버 ➡️ 클라이언트  
  ➔ 요청받은 데이터를 찾아서 건네주는 것
  > 📦 서버: "여기 게시글 10개요"

## 3. API 키 필요성 및 보안 주의사항

### 🔑 API 키가 필요한 이유

1. 인증 (신분 확인): 서버 입장에서 요청을 보낸 사람이 누구인지 확인하기 위함
2. 권한 제어: 유료 사용자만 쓸 수 있는 기능인지, 혹은 관리자인지 구분
3. 트래픽 관리: 한 사람이 너무 많이 요청해서 서버를 다운시키는 것을 막는다

### 🛡️ 보안상 주의사항

- 비밀번호 취급: 남에게 절대 보여주면 안 된다
- 업로드 금지: GitHub 같은 공개된 곳에 코드를 올릴 때 키가 포함되지 않도록 주의
- 환경변수 사용: 코드에 직접 적지 말고(`.env`), 컴퓨터의 환경변수에 숨겨서 사용해야 안전
